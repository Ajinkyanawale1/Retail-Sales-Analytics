# Retail Sales Analytics using SQL

## 📊 Project Overview

This project demonstrates an end-to-end **Retail Sales Data Analysis** workflow using **SQL Server**. The goal is to clean raw retail data, design a **star schema data model**, and perform **exploratory data analysis (EDA)** to generate meaningful business insights.

The analysis focuses on key business metrics such as total sales, customer value, product performance, and regional sales distribution.

---

## 🧰 Technologies Used

* **SQL Server**
* **T-SQL**
* Data Modeling (Star Schema)
* Window Functions
* Aggregations & Analytical Queries

---

## 🗂 Dataset Description

The dataset contains retail sales transaction data including:

* Order Information
* Customer Details
* Product Details
* Sales Revenue
* Shipping Information
* Geographic Data

Key columns include:

* `ORDER_ID`
* `CUSTOMER_ID`
* `PRODUCT_ID`
* `ORDER_DATE`
* `SHIP_DATE`
* `SALES`
* `CATEGORY`
* `SUB_CATEGORY`
* `REGION`

---

## ⚙️ Project Workflow

### 1️⃣ Data Exploration

Initial inspection of the dataset to understand its structure and contents.

Tasks performed:

* Preview raw data
* Count total records
* Check column data types

---

### 2️⃣ Data Cleaning & Validation

Data quality checks were performed to ensure reliable analysis.

Steps included:

* Identifying **NULL values**
* Detecting **duplicate order-product combinations**
* Validating **date formats**
* Converting text dates into **DATE datatype**
* Checking for **invalid sales values**
* Standardizing text fields using **UPPER()**

---

### 3️⃣ Data Modeling (Star Schema)

A **star schema design** was implemented for efficient analysis.

**Dimension Tables**

* `DIM_CUSTOMERS`
* `DIM_PRODUCTS`

**Fact Table**

* `FACT_SALES`

This structure improves query performance and supports scalable analytics.

## 🔍 Advanced SQL Analysis

### Top 5 Products per Region

Identifies the best-selling products in each region using **window functions (RANK)**.

### Year-over-Year Sales Growth

Compares yearly sales performance using **LAG() window function**.

### Customer Lifetime Value (CLV)

Calculates the total revenue generated by each customer and identifies high-value customers.

### Running Sales Total

Computes **cumulative revenue growth over time** using window aggregation.

### Sales Contribution Percentage

Measures how much each product category contributes to overall revenue.
